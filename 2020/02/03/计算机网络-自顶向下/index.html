    Part 1 概述

    1.1 什么是计算机网络

    宏观角度

     计算机网络 = 通信技术 + 计算机技术

    通信技术基本模型

    计算机网络就是一种特殊的通信技术，信号的发出者是电脑，接受者也是电脑，传输的内容变成了数字信号。

    定义：计算机网络就是互连的，自治的 计算机集合。

    1. 自治：相互独立，无主从关系(众生平等)。
    2. 互联：互联互通，通过通信链路(光纤，铜绞线)相互连接。

    在网络中的计算机叫做主机(host)(端系统 end system)，连接主机的叫通信链路。

    如果主机数量多，相隔距离远，直接用光纤，铜绞线相连，未免有些不切实际。所以，引出交换网络。

    主机直接连到交换网络中，通过使用交换节点(路由器，交换机)，实现通信。

    微观角度(组成细节)

     Internet是什么？

    它是全球最大的互联网络，由多个ISP(Internet Service Provider)网络互连形成的网络。

    主流但不唯一

    所以Interne的特点：

    1. 数以千万记的互联的计算设备的集合。在 Internet中的主机，可以运行各种网络应用。
    2. 通信链路：光纤，铜缆，无线电，卫星……
    3. 分组交换(交换网络中)：转发分组(数据包)，路由器(routers),交换机(switches)

    应用角度(产生的服务)

     互联网为网络应用提供通信服务的通信基础设施。(提供了网络编程接口)

    举个例子：Web应用，Email，网络游戏，电子商务，社交网络……

    我们的学习路线就是，从生活实际中的应用入手，逐层学到组成互联网的底层硬件。

    所以又称，自顶向下方法。

    Part 2 协议(Protocol)

     首先设想一个简单问题，我用一根光纤将两台计算机(主机)直接相连，这两台主机就能进行通信了吗？

    答案显然是否定的。

    计算机很笨，他们不会说话。在计算机网络中的数据交换必须遵守事先约定好的规则，即协议。

    现实中犹如我们的交通系统，系统中不仅得有车(主机)，还得有交通规则(协议)，否则，整个系统很可能就会瘫痪。

    任何通信和信息交换过程都需要规则。

    比如说，两个北京人想开启闲聊模式，就会出现：

    A：您今儿吃了吗？

    B：吃嘞，您呢？

    …

    再比如，询问时间时的固定问法：

    A：您好，不好意思，打扰您一下。

    B：您好。

    A：请问现在几点了？

    B：下午8点30。

    A：谢谢。

    网络通信：

    通信主体：机器

    交换内容：数字化信息

    遵守规则：协议

    定义：网络协议(Network Protocol)，简称协议，是为在进行网络中的数据交换而建立的规则，标准或约定。

    它约定了什么呢？

    反观上述例子，人之间说话存在套话。协议规定了两个人(通信实体)之间所交换信息的格式(这话怎么说)，意义，顺序(说话顺序)以及针对收到信息或发生的时间所采取的动作(Actions)。

    网络协议基本要素
    1. 语法(Syntax)：数据与控制信息的结构和格式，信号电平。
    2. 语义(Semantics)：明确的控制信息的含义做出响应，异常处理(差错控制)。
    3. 时序(Timing)：事件顺序，速度匹配。

    协议规范了网络中的所有发送和接收过程。

    例:TCP,IP,HTTP,Skype

    所以，学好协议很重要！！！

    实现数据从源主机通过网络核心送达至目标主机。（数据交换）

    数据交换

    为什么要进行数据交换？

    假设以下有一个完全无向图，每个蓝色的顶点代表一个主机，我们直接用物理介质将它们全部连接到一起，根据离散数学的知识可知，共需要 N(N-1)/2 条边(N为顶点个数)。

    那么，N个主机，就需要至少N² 条链路进行连接。这就导致了部署成本高，技术实现难度大等问题。

    所以我们引入了一种设备，是上图变成下图的状态。

    但是，这种数据交换方式还是有问题，如果设备数量多，距离交换设备远的话，显然依旧不可行，所以我们把交换设备之间连城交换网络，这样就可以两全其美，一劳永逸了。

    交换设备要具有

    1. 动态转接的功能(从一个输入端口交换至另一个输出端口)
    2. 动态分配传输资源

    数据交换的类型

    1. 电路交换
    2. 报文交换
    3. 分组交换
    电路交换

    2. 通信

    多路复用技术

    多路复用(multiplexing),简称复用，是通信技术中的基本概念。它的目的是可以使同一信道在同一时间内被多个用户共享使用。可以提高信道的利用效率

    基本原理：

    1. 将链路/网络资源(带宽)化分成”资源片“
    2. 将资源片分配给各路“呼叫”(call)
    3. 每路呼叫独占分配到的资源片进行通信
    4. 资源片可能闲置(idle)

    典型多路复用方法：

    1. 频分多路复用(Frequency division multiplexing-FDM)
    2. 时分多路复用(Time division multiplexing-TDM)
    3. 波分多路复用(Wavelength division multiplexing-WDM)
    4. 码分多路复用(Code division multiplexing-CDM)
    频分多路复用FDM

    生活实例：电视

    频分多路复用的各用户占用的是不同的带宽资源(单位：Hz不是数据的发送速率)

    用户被分配到一定的频带上，此后的通信过程始终独占这条频带。

    时分多路复用TDM

    时分复用是将时间划分成一段段等长的时分复用帧(TDM帧)，每个用户在每个TDM帧中占用固定序号的时隙。但在频率上不进行分割。

    波分多路复用WDM

    波分复用实际上就是光的频分复用，毕竟我们一般关注的都是光的波长。

    码分多路复用CDM

    广泛用于无线链路共享(卫星通信)

    每个用户分配一个唯一的m bit 码片序列(chipping sequence),为方便讨论，其中0表示为“-1”，1为“1”

    各个用户采用相同的频率载波，利用各自码片编码数据

    编码信号 = (原始数据) * (码片序列)

    如果发送了比特 1(+1)，则发送了自己的 m bit 码片序列

    如果发送了比特 0(-1)，则发送了该码片序列的m bit 码片序列的反码

    为了不在信道中产生冲突，这个码片序列需要相互正交(orthogonal)

    字母 i,j代表两位用户，m是码片长度，S是码片序列，“-”是码片序列的反码。只有用户自己和自己做内积时会得1，其他情况均不得1。

    报文交换(message switching)

    报文是什么？报文就是信息的整体。比如：一张图片，一个文件

    早期应用：电报

    工作机制：

    分组交换(package switching)

    分组：将报文拆解成一系列相对较小的数据包。

    分组交换的过程是拆分 + 重组，那么当主机把分组发至路由器时，路由器需要完整保存分组，然后再转发出去。

    报文交换和分组交换均采取存储-转发机制，仅仅是数据单位的问题。那么交换机制一样，那种数据交换方式更好呢？

    讨论

    为了方便讨论，我们只关注传播延迟(在链路上，从发送的第一个bit开始到到达目的地的结束时间)，且不考虑分组头

    报文交换

    源主机发送了报文至第一个路由器，经历时间 t1 = M/R = 5.0（秒）

    第一个路由器发送到第二个路由器，经历时间 t2 = M/R = 5.0（秒）

    第二个路由器发送到目的主机，经历时间 t3 = M/R = 5.0（秒）

    合计时间 15（秒），同时思考一下，基于存储-转发机制，路由器缓存最少多大？（7.5Mbits），那要是发送一个大小为1Gbits的报文，怎么办？

    分组交换

    由于把整个报文分成了5000个，分组大小为1500bits的分组。

    编号0001的分组经过L/R = 0.001(秒)[1毫秒]后，到达第一个路由器

    第2毫秒，编号0001的分组到达了第二个路由器，编号0002的分组到达了第一路由器

    第3毫秒，分组0001到达目的地，分组0002到达第二路由器，分组0003到达第一路由器

    …

    第4998毫秒，分组4998到达第一路由器

    第4999毫秒，分组4999到达第一路由器，分组4998到达第二路由器

    第5000毫秒，分组5000到达第一路由器

    2毫秒后，第5002毫秒，分组5000到达目的地，总时间5002毫秒||5.002秒

    同理思考一下，基于存储-转发机制，路由器缓存最少多大？（1500bits）

    分组交换报文交付时间

    跳步数 = 路由器数 + 1

    T = 报文传播时间 + 最后一个分组到达的时间

    由于是求最短时间，所以选择跳步数少（所经路由器数目少）的路径，就分组而言，可以将其看成分组头 + 真实数据，即真实数据所需要的分组个数为980 000 / (1000-20) = 1000（个）,整个报文大小就变成了

    980 000 + 20*1000 = 1 000 000 B = 8 000 000 bits (1Byte = 8 bits)

    基于公式，T = 报文传播时间 + 最后一个分组的传播时间

    得出 T = (8 000 000 /100 000 000) + (2*1000*8/100 000 000) = 0.08016(Sec) = 80.16ms

    分组交换和电路交换的比较

    “活动”：存在网络交换的时候是活动的，比如，加载网页时是“活动的”，而浏览已加载网页时，是“空闲的”。

    基于电路交换，使用多路复用技术后，多个用户可以同时使用同一条链路。由于电路交换的本身机制，只要用户不结束网络连接，哪怕只是在看已经加载好的网页，在链路中也是处于“占线”的状态，也就是说哪怕是在空闲状态链路中也不会释放带宽资源，用户独占其带宽，造成资源浪费。

    而分组交换不存在这种问题，完全可以利用用户的空闲时间，假设现在共有35位用户，10位用户同时在线的概率为多大呢？

    这是一个简单的二项分布问题

    k = 35，n = 10，p = 0.1

    可算出10人同时在线的概率为 0.13%

    可算出11人同时在线的概率为 0.033%

    即，实际同时在线人数概率随着假设同时人数的增加而降低，

    若假设35人同时在线，这个概率基本上为 0.00%

    得出的结论是，分组交换可以允许更多用户同时使用网络-资源共享率高

     如同描述一个算法可以看其时间，空间复杂度一样，计算机网络的性能优劣也需要一些指标来进行评价，方便以后进行讨论。所以，本节内容着重讨论如何描述计算机网络的性能，以及相关指标。

    速率

    速率即数据率(data rate)或称数据传输速率，比特率(bit rate)

    含义：单位时间(s)传输的数据量(bit)

    单位：b/s (bps)，kb/s，Mb/s,Gb/s

    换算标准：1000 k = 1 M, 1000 M = 1G

    速率往往是额定速率(理想速率)

    带宽(bandwidth)

    原意：信号的最高和最低频率之差，单位 赫兹(Hz)

    计算机网络中：数字信道所能传送的最高数据传输速率。

    单位：b/s (bps)，kb/s,Mb/s

    延迟/时延(delay/latency)

    全面延迟请看下图

    ①结点处理延迟(nodal processing delay)

    1. 路由器本身要对分组进行检查
    2. 确定输出链路
    3. 基本时间 < 毫秒(ms)

    ②排队延迟(queueing delay)

    1. 分组在路由器缓存中等待的时间
    2. 不确定性高，取决于路由器的拥塞程度

    R：链路带宽(bps) L：分组长度(bits) a:平均分组到达速率

    流量强度(traffic intensity) = La/R

    La/R ~ 0：平均排队延迟很小

    La/R ~> 1：平均排队延迟很大

    La/R > 1：平均排队延迟无限大

    ③传输延迟(transmission delay)

    1. 分组长度： L (bits)
    2. 链路带宽： R (bps)
    3. d = L/R

    ④传播延迟(propagation delay)

    1. d:物理链路长度
    2. s:信号传播速度(2*10e8 m/s)
    3. d(prop) = d/s

    注意传输延迟和传播延迟的区别：

    1. 传输延迟是发送一个分组所需时间。
    2. 传播延迟是信号在数据链路上传播的时间。

    例子：

    车(比特)，车队(分组)，收费站(路由器)

    传输延迟：一个车队过收费站所需时间。

    传播延迟：一辆车从一个收费站到另一个收费站所需的时间。

    总延迟：d (nodal) = d (proc) + d (queue) + d(trans) +d(prop)

    时延带宽积

    时延带宽积 = 传播时延 * 带宽

    新角度解析：带宽单位 b/s，传播时延 s 乘积单位 = bit

    即：整个链路中能容纳的比特数量(以比特为单位的带宽长度)

    分组丢失(丢包)

    路由器队列缓存有限，分组到达已满队列将被丢弃。

    丢包率 = 丢包数 / 已发分组总数

    吞吐量(Throughput)

    表示在发送端和接收端之间的数据传输速率(b/s)

    1. 及时吞吐量(瞬时速率)：给定时刻下的速率
    2. 平均吞吐量：一段时间下的速率

    吞吐量取决于瓶颈链路(bottleneck link)

    Cookie技术

    由于HTTP协议是无状态的，所以有的时候这种特性会给人带来麻烦，比如：当你在网上购物的时候，花了数个小时甄别出5件商品放入购物车时，由于HTTP协议的无状态性，他只会记录最后一条购物车信息，这就会让人抓狂，真是赔了夫人又折兵。

    而Cookie技术的出现解决了HTTP协议“记性差”的问题。

    Cookie的作用：让服务器可以辨别用户身份。注：Cookie通常存在本地且Cookie信息由浏览器保存。Web应用的服务器会为这些数据建库。

    Cookie组件的位置：追加在HTTP响应/请求消息的头部行

    工作原理：

    假设，你在使用Amazon进行购物

    1. 第一次客户端对服务器进行HTTP请求，服务器检索数据库发现没有此用户，在返回原定响应消息的同时，在响应信息的头部行追加了 Set-cookie：1678字段
    2. 在客户端处，存储这个Cookie。第二次访问，在请求信息头部行带上给过的Cookie，服务器查询到了次Cookie，这样就可以通过这个Cookie，找到存储在数据库里的相关信息【购买记录，登陆密码，登陆状态…】，从而返回特定的页面。
    3. 此后无论间隔多久，只有Cookie没失效，就可以一直保存状态。

    Cookie的用途：

    1. 个性化推荐
    2. 身份认证
    3. 购物车

    Part 7 传输层(运输层)

    7.1 运输层概述

    运输层是为它上面的应用层提供服务通信服务的，它通信的范围是两台不同主机中不同进程间的通信。不像接下来的网络层，是在不同主机之间保持通信。

    在运输层上，主要有两种协议，一种是面向报文的UDP协议，另一种是面向连接的TCP协议。

    上图中主机A运行着AP1，AP2两个进程，它们分别占据计算机内两个端口号，同时使用TCP作为运输层协议。那么就会出现协议的复用（multiplexing） [不同进程可以使用相同协议传输数据] 和分用（demultiplexing）问题。

    所以，从上图我们不难看出$\color{red}{网络层是为不同主机之间提供逻辑通信，而运输层为应用进程之间提供端到端的逻辑通信}$。

    因为网络层及其以下的网络都是不可靠的，只有采用了TCP协议时，才会建立起一个可靠的全双工信道。

    7.1.1 传输层的两个重要协议

    用户数据报控制协议 UDP (User Datagram Protocol)

    传输控制协议 TCP (Transmission Control Protocol)

    TCP在传输层的传输单位是TCP报文段，UDP的为UDP报文或用户数据报。

    特点概述

    UDP：一种无连接协议，提供无连接服务。在传送数据之前不需要先建立连接。
    传送的数据单位协议是 UDP 报文或用户数据报。
    对方的运输层在收到 UDP 报文后，不需要给出任何确认。
    虽然 UDP 不提供可靠交付，但在某些情况下 UDP 是一种最有效的工作方式。（手机实时游戏，视频会议）

    TCP：一种面向连接的协议，提供面向连接的服务。
    传送的数据单位协议是 TCP 报文段 (segment)。
    TCP 不提供广播或多播服务。
    由于 TCP 要提供可靠的、面向连接的运输服务，因此不可避免地增加了许多的开销。这不仅使协议数据单元的首部增大很多，还要占用许多的处理机资源。

    7.1.2 端口

    由于传输层协议需要知道最后的数据要发给哪个应用进程，操作系统预留了16位的端口号，以便应用进程使用。这就相当于门牌号，路由器通过IP地址寻找主机，传输层协议通过端口号寻找目标进程。我们有两大类端口号：

    (1) 服务器端使用的端口号
    熟知端口，数值一般为 01023。系统默认规定的端口号，有特殊功能。
    登记端口号，数值为 1024    49151，为没有熟知端口号的应用程序使用的。使用这个范围的端口号必须在 IANA 登记，以防止重复。比如MySQL常见的3306
    (2) 客户端使用的端口号
    又称为短暂端口号，数值为 49152~65535，留给客户进程选择暂时使用。
    当服务器进程收到客户进程的报文时，就知道了客户进程所使用的动态端口号。通信结束后，这个端口号可供其他客户进程以后使用。

    7.2 用户数据报协议 UDP

    7.2.1 UDP协议概述

    主要特点 ※

    (1) UDP 是无连接的，发送数据之前不需要建立连接，，因此减少了开销和发送数据之前的时延。
    (2) UDP 使用尽最大努力交付，即不保证可靠交付，因此主机不需要维持复杂的连接状态表。
    (3) UDP 是面向报文的。UDP 对应用层交下来的报文，既不合并，也不拆分，而是保留这些报文的边界。UDP 一次交付一个完整的报文。

    (4) UDP 没有拥塞控制，因此网络出现的拥塞不会使源主机的发送速率降低。这对某些实时应用是很重要的。很适合多媒体通信的要求。

    (5) UDP 支持一对一、一对多、多对一和多对多的交互通信。
    (6) UDP 的首部开销小，只有 8 个字节，比 TCP 的 20 个字节的首部要短。

    我们先忽略伪首部，先看首部字段，4个字节，看图说话，很好理解。其中长度为整个UDP数据报长度，检验和初始值为0，经过和伪首部运算之后得出结果后填入。

    7.3 传输控制协议 TCP

    7.3.1 TCP协议概述

    （1）TCP 是面向连接的运输层协议。
    （2）每一条 TCP 连接只能有两个端点 (endpoint)，每一条 TCP 连接只能是点对点的（一对一）。
    （3）TCP 提供可靠交付的服务。
    （4）TCP 提供全双工通信。
    （5）面向字节流
    TCP 中的“流”(stream)指的是流入或流出进程的字节序列。
    “面向字节流”的含义是：虽然应用程序和 TCP 的交互是一次一个数据块，但 TCP 把应用程序交下来的数据看成仅仅是一连串无结构的字节流。将结构化的数据转换为0101这样的字节，以字节的形式发送。

    每次TCP协议发送的字节流的大小会根据当前网络拥堵情况和窗口期自动调整大小，所以会出现上图中发送数量不同的情况

    7.3.2 TCP连接

    为了达到可靠传输这一需求，TCP把连接作为最基本的抽象。其实很好理解，为了让两台主机可靠的传输，我们会以逻辑的形式构建一条虚链路，TCP协议连接的端点既不是IP也不是端口而是套接字（Socket）

    $$
    套接字 socket = (IP地址 : 端口号)\
    TCP 连接 ::= {socket1, socket2} = {(IP_1: port_1)，(IP_2: port_2)}
    $$

    同一个 IP 地址可以有多个不同的 TCP 连接。(同一台服务器上的MySQL可以被不同主机访问)
    同一个端口号也可以出现在多个不同的 TCP 连接中。（不同主机访问一台服务器上的一个MySQL进程）

    7.3.3 TCP可靠传输原理

    停止等待协议

    理想的传输条件有以下两个特点：

    (1) 传输信道不产生差错。
    (2) 不管发送方以多快的速度发送数据，接收方总是来得及处理收到的数据。

    在这样的理想传输条件下，不需要采取任何措施就能够实现可靠传输。
    然而实际的网络都不具备以上两个理想条件。必须使用一些可靠传输协议，在不可靠的传输信道实现可靠传输。

    “停止等待”就是每发送完一个分组就停止发送，等待对方的确认。在收到确认后再发送下一个分组。
    全双工通信的双方既是发送方也是接收方。
    为了讨论问题的方便，我们仅考虑 A 发送数据而 B 接收数据并发送确认。因此 A 叫做发送方，而 B 叫做接收方。

    那么在通信时便会有如下情况发生：

    1 无差错

    2 有差错

    有差错我们就采用超时重传机制，因为发送时的差错无外乎就是数据校验不通过（分组错误），数据包丢失（分组丢失）。

    再看确认过程，也有可能出错，一个是确认丢失，另一个是确认迟到。很好理解，前者是B的回执丢失了，后者是B发送的回执超过了A的等待时间了。

    如果发生确认丢失，则会进行如下操作：

    连续ARQ协议

    通常主机 A 最终总是可以收到对所有发出的分组的确认。如果 A 不断重传分组但总是收不到确认，就说明通信线路太差，不能进行通信。使用上述的确认和重传机制，我们就可以在不可靠的传输网络上实现可靠的通信。
    像上述的这种可靠传输协议常称为自动重传请求 ARQ (Automatic Repeat Request)。意思是重传的请求是自动进行的，接收方不需要请求发送方重传某个出错的分组。

    既然SOH和EOT都是编码，那么一定会存在一种情况，那就是装在数据帧内的数据和这两个特殊编码相同，此时我们就要类比编程中的转义字符，对帧进行字节填充（字符填充），在这些特殊字符前加上ESC（16进制数为1B,2进制 00011011）。在原始数据中只要出现特殊编码字符，我们就在它前面加上ESC，到达目标主机后，删除从左向右数，所有第一次出现的ESC。

    在数据传输过程中，很有可能受到干扰，导致传输的数字比特流0变成1，1变成0。造成信息的传输错误。信噪比越高，信号质量越好。我们为了能够在接收端检验发送的数据是否完全正确时。接下来我们就要使用的就是循环冗余检验CRC(Cyclic Redundancy Check)用于检错。

    9.2 点对点协议（PPP协议）

    在前面，我们初步学习了数据链路层协议所面临的三种问题（封装成帧，透明传输，差错检测）的解决方法，现在，我们给出一个常用的协议来系统地规定数据链路层上的信息交换规则。

    9.2.1 PPP协议的功能

    以下内容理解即可。

    (1) 简单：在同一条物理链路上进行点对点的数据传输，对数据链路层的帧不进行纠错，不需要序号，不需要流量控制。

    (2) 封装成帧：加入帧界定符。

    (3) 透明性：字节填充法。

    (4) 多种网络层协议：在同一条物理链路上同时支持多种网络层协议（如IP和IPX等）的运行。

    (5) 多种链路类型：PPP必须能够在多种类型的链路上运行，例如串行或并行链路。

    (6) 差错检测：接收方收到一个帧后进行CRC检验，若正确就收下这个帧，反之则丢弃。

    (7) 检测连接状态：自动检测链路是否处于正常工作状态。

    9.2.2 PPP协议的组成

    (1) PPP协议提供一个将IP数据报封装到串行链路的方法。IP数据报在PPP帧中就是信息部分，长度受最大传送单元MTU的限制。PPP支持异步链路（无奇偶校验的8比特数据）和面向比特的同步链路。

    (2) 链路控制协议LCP（Link Control Protocol）

    当线路不再需要时，挑出这些线路，测试它们，商议选择，并仔细地再次释放链路控制协议。

    (3) 一套网络控制协议NCP（Network Control Protocol）

    其中每一个协议支持不同的网络层协议，如IP、OSI的网络层等。

    9.2.3 PPP帧的格式

    首部包含四个字段（注意，字段不等于字节）

    字段 长度 说明
    F 1 固定值为0x7E，代表帧开始或结束
    A 1 固定值为0xFF，无特殊含义
    C 1 固定值为0x03，无特殊含义
    0x0021——信息字段是IP数据报
    0xC021——信息字段是链路控制数据LCP
    0x8021——信息字段是网络控制数据NCP
    协议 2 0xC023——信息字段是安全性认证PAP
    0xC025——信息字段是LQR
    0xC223——信息字段是安全性认证CHAP

    中间为信息部分,一般会限定大小，上图是不超过1500字节。

    尾部包含两个字段

    字段 长度 含义
    FCS 2 帧检验序列，用于校验
    F 1 固定值为0x7E，代表帧开始或结束

    当PPP协议帧中出现特殊字符时，根据链路的不同会选择不同的处理方式。

    字节填充法

    (a) 当信息字段中出现0x7E时，将每一个0x7E字节转变成2字节序列（0x7D，0x5E）。

    (b) 若信息字段中出现一个0x7D的字节，则将其转变成2字节序列（0x7D，0x5D）。

    (c) 若信息字段中出现ASCII码的控制字符(即数值小于0X20的字符)，则该字符前面需要加入一个0x7D字节，同时将该字符的编码加以改变(+20)。例如，出现0x03，就要把它转为2字节序列(0x7D，0x23)

    0比特填充法

    PPP协议用在SONET/SDH链路时，使用同步传输（一连串的比特连续传送），此时使用0比特填充法。

    (a) 在发送端扫描整个信息字段，当有5个1连续出现时，立即填入一个0，保证信息字段中不会出现6个连续1。

    (b) 在接收端删除连续5个1后面的0。


    9.3 使用广播信道的数据链路层

    上节提到的数据链路层协议是针对两台主机直连时的协议组织方式。现在我们换一种场景来考虑这个问题。

    假设在机房有一个局域网，此类网络的特点是网络为一个单位所拥有，且地理范围和站点数目皆有限。我们使用一根总线将所有主机串联起来形成总线网。

    主机和外界局域网的连接是通过适配器(adapter)进行的，日常生活中我们也将其称之为网卡，它负责接收和处理各种帧。

    9.3.1 CSMA/CD协议

    由于我们讨论的以太网（总线型局域网），采用的是一根总线在同一时刻只能发送或接收数据，所以采用CSMA/CD协议的通信是典型的半双工通信。那么此时就会产生一个很容易想到的问题如果在同一时间，多个主机同时发送数据，就会互相干扰，导致所有的数据均变成无效数据，此时就需要一种协议来检测数据碰撞和重发数据帧。这就是CSMA/CD协议，中文名称为载波监听多点接入/碰撞检测协议(Carrier Sense Multiple Access with Collision Detection)。

    它要解决的是如下3个问题：

    1. 多点接入:在总线型网络中，许多计算机以多点接入的形式连接在一起。
    2. 载波监听:使用电子技术检测总线上是否有其它主机正在发送数据。无论是在发送前，还是发送时每个主机都要检测通道
    3. 碰撞检测: 边发送边监听，如果总线上的电压超过了阈值（传输的信号发生了叠加），那么也就意味着总线上至少有两台主机在同时发送数据

    如上图，假设主机A到主机B的单程传输时间为τ 秒，那么主机A要想知道是否发生碰撞的最小时间为2τ 秒（如果A在发送完数据后2τ时间间隔之内接收到了数据，那么一定发生了碰撞）。我们将这个2τ的时间称为争用期，如果在争用期内没有检测到碰撞，那么之后的数据一定不会发生碰撞，如果发生了碰撞，此时我们就要采用截断二进制指数退避算法来规避一个随机时间。

    截断二进制指数退避算法

    设基本争用期为2τ ，具体的争用期为51.2μs，10Mbits的以太网，争用期内可以发送512bit，64字节。

    规定一个离散的集合，如下：
    $$
    set = {0,1,3,…,(2^k-1)}\
    k = min(重传次数,10)
    $$
    我们的退避时间就是从这个集合中，随机取出一个数 r，让主机等待*2τ \ r μs **

    当重传次数达到16次时，此时丢弃数据，不再发送数据并向上层报错。

    所以，一旦发生碰撞，就一定是在发送前64个字节之内产生的，因此我们将其称为最短有效帧

    Part 10 物理层

    10.1 物理层的意义和特性

    物理层想解决的问题

    物理层想解决的问题是，如何才能在各种计算机的传输媒体上传输数据比特流。即$\color{red}{屏蔽掉不同传输媒介和通信手段的差异，让数据链路层感受不到这些差异。}$

    同时，物理层的协议也常称为规程(procedure)。

    物理层的特性
    1. 机械特性：指明接口所用的接线器的形状，尺寸。即，统一硬件的物理标准。
    2. 电气特性：指明在接口电缆各线上出现的电压范围。即，统一电压范围。
    3. 功能特性：指明接口电缆上出现某一电平的意义。类似协议中的语义特性。
    4. 过程特性：指明对不同功能的各种可能事件的出现顺序。类似协议中的时序特性。

    10.2 数据通信的基本知识

    物理层是最最接近硬件的一层，同时也是跨学科幅度最大的一层。想学好物理层的知识，必须对通信原理有一定的了解。所以我们开始介绍数据通信系统的相关知识。

    10.2.1 数据通信系统的模型

    一个通信系统可以划分为3大部分，即源系统，传输系统，目的系统，如下图：

    源点，又称信源，它是产生要传输数据的设备。

    发送器，将信源发出的数字比特流通过调制器编码为信号进行传输。

    接收器，将信号通过解调器转换为数字比特流。

    终点，接收数字比特流，进行下一步的操作。

    通信的目的是传递信息(Message)。如，文字，语音，图片，视频等。在计算机的世界里，一切信息都要以数据(Data)的形式进行表示。即，$\color{red}{数据是运输信息的实体}$。数据的本质就是比特流（二进制数组成的串），这种比特流，在通信链路中通常需要通过调制解调器，将其变成信号（Signal）。信号分为两大类。分别是

    1. 模拟信号，特点是连续性
    2. 数字信号，特点是离散性


    10.2.2 信道(Channel)

    信号向某一方向传输的介质是信道。按照方向，可分为3大类：

    1. 单向通信（单工通信）：只能有一个方向的通信，不能进行交互。如，收音机。
    2. 双向交替通信 （半双工通信）：同一时刻内，只能存在一个方向上的通信。如，对讲机。
    3. 双向同时通信（全双工通信）：同一时刻内，双方可以同时发送和接收信息。如，打电话。

    来自信源的信号常称为基带信号（基本频带信号）。比如人说话时的语音就是基带信号，它不能被信道直接传输，所以需要对基带信号进行调制（modulation）。举例：人的语音不能直接放到光纤里传输，需要将其编码后才可以进行传输。

    基带信号调制后，它还是基带信号，但是它不能传播很远，所以我们还需对其进行基带调制，使其可以在信道上传播很远。通常我们将数字信号之间的基带调制称之为编码（Coding），模拟信号通过载波的调制称为带通调制。

    常用的编码方式 ※
    • 不归零制：正电平代表 1，负电平代表 0
    • 归零制：正脉冲代表 1，负脉冲代表 0
    • 曼彻斯特编码：位周期中心，上跳为 0 （低电平->高电平），下跳为 1 （高电平->低电平）
    • 差分曼彻斯特编码：1. 在每一位中心处始终有跳变（强制性）2. 位开始边界处有跳变为 0 ，没有跳变代表 1
