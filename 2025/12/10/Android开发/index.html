1.1 Kotlin 编译器 (本节可略过)

由于Kotlin语言及其生态的更新迭代，因此产生了Kotlin编译器。

下面是K2 Kotlin 编译器前端图

这个编译器带来了如下优势：

1.改进了调用解析和类型推断。编译器的行为更加一致，也能更好地理解你的代码。

2.为新语言特性更容易引入语法糖。将来，当引入新功能时，你将能够使用更简洁、可读的代码。

3.更快的编译时间。编译时间可以大大加快。

4.增强的IDE性能。从2025.1开始，IntelliJ IDEA使用K2模式来分析你的Kotlin代码，增加稳定性并提供性能改进。有关更多信息，请参阅ide中的支持。

上述内容的重点在于Kotlin 是一种编译型语言，他在编译过程中会将代码转换成中间字节码(通常是Java 字节码)， 以便在JVM上运行。此外，Kotlin 也支持将代码编译为 JavaScript 和原生代码（使用 Kotlin/Native），使其能够在多种平台上运行。

总结来说，Kotlin 是编译型语言，并且具有跨平台的能力。

在Kotlin语言中，所有变量和数据类型都拥有一个类型，在Kotlin中变量的类型和C++语言中一样重要，因为编译器需要知道针对此数据类型，用户可做的操作有哪些，换句话说就是该类型有哪些方法和属性。

Kotlin提供了内置的类型推导功能，因此类似名称如number的变量，编译器会默认把它推导成Int类型：

那么既然可以当作数字，在Kotlin中，自然也支持增强赋值 (Augmented assignments﻿)功能，即类似C++中的运算操作符:

+=, -=, *=, /=, 和 %=

在此，请特别注意，在Kotlin中上述操作不是运算表达式，而是增强赋值，因为编译器对上述语法的操作如下：

0x01 增强赋值表格
Expression Translated to
a += b a.plusAssign(b)
a -= b a.minusAssign(b)
a *= b a.timesAssign(b)
a /= b a.divAssign(b)
a %= b a.remAssign(b)

从此可以看出，Kotlin语言和Java有着很深的渊源，其底层思想都是以面向对象为方向发展的，而不像C++一样底层到根据数据类型，让编译器判断需要申请多少字节的数据。在这个层面上，Kotlin语言对人更加友好，程序员无需操心内存和数据类型之间的关系。

2.3.3 组合 (The Composition)

现在学习的Compose是一种声明式UI框架，这意味着，你通过代码声明了UI界面的样子。如果你想让账单的初始值为100，那么你就给组合函数里面传个100就行。

当软件运行过程中或用户交互软件时导致UI界面发生更改时，会发生什么？

举例来说，如果你希望amountInput的值会随着用户输入而发生更新。那时，你依赖一个称为重新组合（recomposition）的过程来更新应用的组合（Composition）。

组合（Composition）是 Compose 执行可组合函数时构建的 UI 描述。Compose 应用调用可组合函数将数据转换为 UI。如果发生状态变化，Compose 会使用新状态重新执行受影响的可组合函数，从而创建一个更新的 UI——这称为重新组合（recomposition）。Compose 会为你安排重新组合的过程。

当Compose第一次执行可组合函数的时候，此时称为初始化组合 (initial composition), ，它会记录下你调用的可组合函数，以描述 UI 的组合。重新组合 (recomposition)是在响应数据变化时，Compose 重新执行可能已更改的可组合函数，并更新组合以反映任何变化。

组合只能通过初始组合生成，并通过重新组合进行更新。修改组合的唯一方式是通过重新组合。为此，Compose 需要知道要跟踪的状态，以便在接收到更新时安排重新组合。

在本例中，就是 amountInput 变量，因此每当其值发生变化时，Compose 就会安排一次重新组合。

在 Compose 中，你使用 State 和 MutableState 类型使应用中的状态可被 Compose 观察或跟踪。State 类型是不可变的，因此你只能读取其中的值，而 MutableState 类型是可变的。你可以使用mutableStateOf() 函数创建一个可观察的 MutableState。该函数接受一个初始值作为参数，并将其包装在一个 State 对象中，从而使其值变为可观察。

其中，mutableStateOf()函数的返回值MutableState具有如下特点:

  • 保存状态，在本例中表示用户输入的账单金额
  • 是可变的, 表示其值可发生变化
  • 是可被观察的，表示Compose观察其值的变化，并通过重新组合更新UI

0x01 理解有状态和无状态可组合函数 (Understand stateful versus stateless composables)

在如下情况下，应当考虑状态提升:

  • 在多个组合函数中需要共用(分享)的状态
  • 创建一个应用中可复用的无状态组合函数

当你从一个可组合函数中提取出状态时，意味着你将状态管理的责任转移到了外部，这样可组合函数本身就不再持有任何状态。进而，该可组合函数就变成了无状态可组合函数。

换而言之，所谓的组件提升，其实就是把通过var xxx by remeber {mutableStateOf(…)} 创建的状态和与该状态相关的函数转移到这个可组合函数的调用者中，让这个可组合函数变成无状态可组合函数。
